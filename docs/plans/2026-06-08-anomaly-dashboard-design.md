# 异常检测分析看板设计文档

## 概述

构建一个高性能的交互式异常检测分析看板，用于可视化分析时序异常检测模型的输出结果。

## 技术栈

- **后端**: Python FastAPI
- **前端**: 原生 HTML5 + Tailwind CSS CDN + 原生 ES Module (Vanilla JS)
- **可视化引擎**: Apache ECharts (CDN)

## 项目结构

```
/mnt/sda/cjmProject/VETime/
├── app.py                 # FastAPI 后端 (~150 行)
└── static/
    ├── index.html         # 页面结构 (~80 行)
    ├── style.css          # Tailwind CDN + 自定义样式 (~50 行)
    └── app.js             # ECharts 图表逻辑 (~400 行)
```

## 后端设计

### 启动命令

```bash
python app.py --data-dir ./output/VETime --port 8000
```

### API 接口

| 接口 | 方法 | 功能 | 返回示例 |
|------|------|------|---------|
| `/` | GET | 返回前端页面 | HTML |
| `/api/datasets` | GET | 获取数据集列表及样本 | `{"NAB": ["id_1", ...], "WSD": [...]}` |
| `/api/data/{dataset}/{sample_id}` | GET | 获取样本数据 | `{"value": [...], "label": [...], "anomaly_score": [...]}` |

### 数据解析规则

- 文件名格式：`{序号}_{数据集}_id_{样本号}_{类别}_tr_{长度}_1st_{编号}_results.pkl`
- 样本 ID：提取 `id_X` 部分作为唯一标识
- 数据集：提取大写缩写（NAB、WSD、UCR、YAHOO、TODS、MGAB、NEK、SED、IOPS）

### 错误处理

- 数据目录不存在：启动时报错退出
- 文件名解析失败：跳过该文件，日志记录警告
- pkl 文件损坏：返回 500 + 具体错误信息
- 样本不存在：返回 404

## 前端设计

### 页面布局

```
┌─────────────────────────────────────────────────────────────┐
│  Header: 异常检测分析看板                                      │
├──────────┬──────────────────────────────────────────────────┤
│  侧边栏   │                 主看板区                          │
│  (240px) │                                                   │
│          │  ┌─────────────────────────────────────────────┐ │
│  数据集   │  │ 图一：时序与异常分数对齐图                     │ │
│  [下拉]   │  │ (双Y轴 + markArea + dataZoom)               │ │
│          │  └─────────────────────────────────────────────┘ │
│  样本     │  ┌─────────────────────────────────────────────┐ │
│  [下拉]   │  │ 图二：区分度分析直方图                        │ │
│          │  │ (正常/异常分数分布对比)                       │ │
│          │  └─────────────────────────────────────────────┘ │
│          │  ┌─────────────────────────────────────────────┐ │
│          │  │ 图三：关联性散点图                            │ │
│          │  │ (abs(value) vs anomaly_score)               │ │
│          │  └─────────────────────────────────────────────┘ │
└──────────┴──────────────────────────────────────────────────┘
```

### 侧边栏组件

- 数据集下拉框：动态加载可选数据集（9 个数据集）
- 样本下拉框：根据数据集动态过滤样本
- 加载提示：数据请求时显示 spinner

### 图表规格

#### 图一：时序与异常分数对齐图

- X 轴：时间步 (0 ~ N)
- 左侧 Y 轴：原始信号 value（折线图）
- 右侧 Y 轴：异常分数 anomaly_score（半透明面积图）
- 背景标注：label=1 的连续区间用 markArea 标红
- 交互：dataZoom 滑块 + 鼠标滚轮缩放 + 拖拽平移

#### 图二：区分度分析直方图

- 类型：双色重叠直方图
- 实现：前端计算分箱（50 bins）
- 正常组（label=0）：蓝色半透明
- 异常组（label=1）：红色半透明

#### 图三：关联性散点图

- X 轴：abs(value)
- Y 轴：anomaly_score
- 交互：支持框选放大和缩放

### 性能优化

- 所有图表启用 `large: true` 和 `progressive: 3000`
- 禁用动画 (`animation: false`)
- 使用 Canvas 渲染器

### 前端错误处理

| 场景 | 处理方式 |
|------|---------|
| 数据集/样本列表加载失败 | 显示错误提示，侧边栏禁用 |
| 样本数据请求失败 | Toast 提示错误信息 |
| 无异常点（label 全为 0） | 图二提示"无异常数据" |
| 超长数据（>100,000 点） | 正常渲染，依靠 ECharts large 模式 |

## 数据源

- 路径：`./output/VETime/`（可通过命令行参数指定）
- 格式：pkl 序列化的 pandas DataFrame
- 列：value (float), label (int: 0/1), anomaly_score (float)
- 数据集数量：9 个（NAB、WSD、UCR、YAHOO、TODS、MGAB、NEK、SED、IOPS）
- 样本总数：约 700 个
- 数据长度范围：576 ~ 150,000 时间步

## 日志输出

- 启动时：打印数据目录、扫描到的文件数量、数据集统计
- 请求时：打印 API 调用日志（可选调试模式）
